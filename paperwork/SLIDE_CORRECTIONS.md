## ⚠️ IMPORTANT — Flan-T5 Mention Is Misleading

**Slide says:** *"Flan-T5 large language model encodes enhanced text embeddings"*

**Reality:** The code (`src/generation.py`) uses **AudioLDM 2's built-in CLAP text encoder**, not Flan-T5.  
Flan-T5 is mentioned in related work / literature but is **NOT used** in our pipeline.

**Fix:** Change to: *"CLAP text encoder encodes enhanced text embeddings"*

## ⚠️ IMPORTANT — FAD Not Implemented

**Slide says (Standard Metrics):** *"FAD – Measures audio realism and perceptual quality"*

**Reality:** FAD is NOT computed anywhere in the codebase (`src/evaluation.py` has no FAD code).  
We only compute POAS (CLAP cosine similarity) + ASR-based semantic score for speech.

**Options:**
- Remove FAD from the slide entirely, OR
- Rephrase as *"FAD is a planned future metric; we currently implement POAS, CRI, CDTS"*

## 📝 MINOR — POAS Score Range

**Slide says:** *"Range: -1 to 1 (higher = better)"*

**Reality:** In our implementation, POAS is **clamped to [0, 1]** using `max(0.0, similarity)`.
Negative cosine similarities are set to 0.

**Fix:** Change to *"Range: 0 to 1 (higher = better)"*

## 📝 MINOR — CRI Formula Description

**Slide says:** *"Formula: std(POAS scores across all samples)"*

**Reality (from `evaluation.py`):** CRI = **mean of per-domain standard deviations** (NOT global std).  
This is a more meaningful metric — it measures within-domain consistency per domain, then averages.

**Fix:** Change to: *"Formula: mean(per-domain std of POAS scores)"*
