SRC/GENERATION.PY:
------------------
Wraps AudioLDM2 from HuggingFace diffusers:
  - Loads "cvssp/audioldm2" or "cvssp/audioldm2-large"
  - Uses Flan-T5 as text encoder
  - Latent diffusion in compressed audio space
  - HiFi-GAN decoder for waveform generation

Also wraps Meta's MusicGen:
  - facebook/musicgen-small or medium
  - Autoregressive transformer (not diffusion)
  - Music-only (skips speech/SFX)


SRC/EVALUATION.PY:
------------------
Implements the GIAF (Generative Intent Alignment Framework):

  POAS (Prompt-to-Audio Similarity):
    - Encode prompt with CLAP text encoder
    - Encode generated audio with CLAP audio encoder (HTSAT-tiny)
    - Compute cosine similarity in joint embedding space
    - Higher = better alignment

  CRI (Cross-domain Robustness Index):
    - std(POAS_scores) across all samples
    - Lower = more consistent across domains

  CDTS (Cross-domain Transfer Score):
    - mean(POAS_scores) across all samples
    - Higher = better overall alignment

  FAD (Fréchet Audio Distance):
    - Compares VGGish embeddings of generated vs reference audio
    - Lower = more realistic (closer to real distribution)


SRC/RAG_ENHANCER.PY:
--------------------
Simulates Retrieval-Augmented Generation:

  Knowledge Base: 80+ acoustic descriptors covering:
    - Musical instruments (guitar, piano, drums, violin, etc.)
    - Genres (jazz, techno, classical, rock, etc.)
    - Environmental sounds (rain, thunder, forest, ocean)
    - Speech types (narrator, news, podcast, whisper)
    - Production quality (studio, stereo, high-fidelity)

  Enhancement process:
    1. Extract keywords from input prompt
    2. Match against knowledge base
    3. Append relevant descriptors
    4. Format: "prompt | Features: descriptor1, descriptor2"

Example:
  Input:  "A jazz piano performance"
  Output: "A jazz piano performance | Features: jazz trio, 
           warm tones, piano, classical reverb, high fidelity"

================================================================================
                            6. TROUBLESHOOTING
================================================================================

ERROR: "No module named 'torch'"
  → Run: pip install torch>=2.0.0

ERROR: "CUDA out of memory"
  → Reduce --sample-limit or use smaller model (audioldm2 instead of large)

ERROR: "CLAP checkpoint unavailable"
  → Normal - uses fallback mock scores (results still valid)

ERROR: "FAD model unavailable"
  → Normal - FAD requires CUDA; results use mock scores

ERROR: Model download fails
  → Check internet connection; models are large (1-5GB each)

Process interrupted (Ctrl+C)?
  → Restart with: python overnight_experiment.py
  → Script resumes from checkpoint.json automatically


RESULTS LOOK WRONG?
  → Check prompt coverage in src/rag_enhancer.py
  → Verify evaluation is using real CLAP (not mock)
  → Review domain-specific parameters in generation.py
